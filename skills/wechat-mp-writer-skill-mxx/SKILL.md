# 微信公众号写作助手

这是一个面向微信公众号内容生产的组合型 skill，负责“写什么、怎么写、怎么更像人写、怎么发出去”。

它不直接替代发布工具，而是与 `wechat-mp-publisher` 配合：
- 本 skill 负责选题、写作、润色、配图建议、发布前整理
- `wechat-mp-publisher` 负责真正调用公众号接口创建草稿 / 发布

## 适用场景

- 用户要从零开始写一篇公众号文章
- 用户已有草稿，需要润色成更适合公众号的版本
- 用户想降低 AI 味，让内容更像真人写的
- 用户写完后要直接进入公众号草稿箱

## 工作模式

### 1. 从零开始写

流程：
1. 明确主题 / 受众 / 目标
2. 给出 3-5 个可写角度
3. 选一个角度展开大纲
4. 输出完整文章
5. 做一轮“去 AI 味”润色
6. 给出配图建议
7. 如用户要求，交给 `wechat-mp-publisher` 发到公众号草稿箱

### 2. 已有草稿，帮我重写

流程：
1. 理解原文意图
2. 保留核心信息
3. 优化结构和节奏
4. 公众号化表达
5. 去 AI 味处理

### 3. 仅润色 / 去味

重点处理：
- 避免模板化连接词
- 避免过于工整的段落结构
- 增加口语感、真实感、停顿感
- 插入更具体的观察和场景
- 保留观点锋芒，不写成空话

### 4. 仅发布

如果用户已经有定稿：
1. 整理出标题、作者、摘要、正文
2. 产出适合发布的 Markdown 或 HTML
3. 调用 `wechat-mp-publisher`

## 写作风格要求

### 技术干货

- 先说结论，再讲过程
- 有实操步骤
- 有踩坑点
- 少空话，多细节

### 故事叙事

- 用场景起笔
- 有情绪变化
- 有细节，不要只写观点
- 结尾要有回味或行动点

### 观点评论

- 立场清楚
- 论据够硬
- 不要假中立
- 能回应反对意见更好

## 去 AI 味规则

润色时，优先做这些事：

- 少用“首先、其次、最后、总之、综上所述”
- 少用过于标准的三段式结构
- 加入口语化表达，但别太油
- 允许句子长短不一
- 允许轻微重复和停顿
- 多写具体观察，少写抽象套话
- 如果适合，可以加入一句很像真人会说的话，比如：
  - “说实话，这里我一开始也踩坑了。”
  - “讲真，很多人卡就卡在这一步。”
  - “你乍一看觉得没什么，真上手就不是那么回事。”

## 配图建议规则

按文章类型给建议：

- 技术教程：代码截图、流程图、效果图
- 产品体验：界面对比图、前后变化图、使用场景图
- 行业观点：数据图表、新闻截图、趋势图
- 个人经历：场景图、对话截图、相关物件图
- 工具推荐：功能截图、操作流程图、封面图

## 发布前整理规则

如果最终要进入公众号：

- 输出一版适合公众号的 Markdown
- 标题不要太长，尽量利于点击
- 摘要控制在短句内，别像论文摘要
- 小标题清晰，层级别太深
- 代码块要标语言
- 图片位置提前标注清楚

## 与 wechat-mp-publisher 的配合

当用户明确说“发到公众号”时：

1. 先整理出：
   - 标题
   - 作者
   - 摘要
   - Markdown/HTML 正文
2. 若缺封面图素材，提醒用户补 `thumb_media_id` 或先只建草稿模板
3. 使用 `wechat-mp-publisher` 创建草稿
4. 除非用户明确要求，不要直接执行最终发布

## 默认输出格式

正常情况下，给用户的结果建议按这个顺序：

1. 标题建议
2. 摘要
3. 正文
4. 配图建议
5. 如果用户要发布，再整理成发布参数

## 注意

- 目标是“更像真人作者”，不是“故意伪装成某个具体真人”
- 可以增强表达个性，但不要捏造事实、经历或数据
- 如果用户提供了自己的真实经历，优先保留那些细节
