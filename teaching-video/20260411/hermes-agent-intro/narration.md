# Hermes Agent 入门视频 Narration

## Scene 01 - 开场：学完能判断要不要试

如果你已经用过 Claude Code、Codex、OpenClaw，可能会发现一个问题：agent 单次执行任务越来越强，但长期使用时，我们还是在反复解释背景、重复交代流程、重复提醒它哪些坑不能踩。今天这条视频，用 3 分钟讲清楚 Hermes Agent：它不是又一个聊天框，而是更像一个长期在线、会沉淀经验的 AI 同事。

## Scene 02 - 它最适合谁

Hermes Agent 最适合三类人先试。第一，你有很多重复工作，想把经验变成可复用 skill。第二，你希望 agent 不只在本机终端里工作，也能从手机、群聊或远程机器继续接任务。第三，你愿意维护一套开源 agent 工作台，而不是只依赖封闭 SaaS 的固定功能。如果只是偶尔问答，它可能有点重。

## Scene 03 - 一次性工具和长期同事

很多 agent 工具像一次性外包：你给任务，它执行，结束。下次再来，它未必记得你的目录结构、审批习惯、发布流程和常见偏好。Hermes Agent 的思路更接近长期同事：它强调长期记忆、过往会话搜索、用户模型，以及从复杂任务中创建和改进 skills。核心价值不是一次做完，而是做过一次之后，下次更容易复用。

## Scene 04 - 四个核心能力

可以把 Hermes Agent 的能力拆成四块。第一是记忆，减少重复解释。第二是 skills，把流程沉淀成工具化经验。第三是跨平台入口，让 agent 不只活在一个终端窗口里。第四是远程运行和后台任务，让长测试、构建、巡检可以在 VPS、Docker 或 SSH 后端继续跑。这四块连起来，才像一个真正长期在线的工作台。

## Scene 05 - 国内环境下怎么开始

在国内环境试 Hermes Agent，不建议一上来就从 Telegram、Discord 或 Google 服务开始。更稳的路线是：先把它当成本机或 VPS 上的开源 agent runtime。模型侧优先接 Kimi、MiniMax、GLM，或者公司内部兼容 OpenAI API 的 endpoint。入口侧，如果团队已经用飞书、钉钉或企业微信，就看它的 gateway 和插件架构能否接进现有协作系统。

## Scene 06 - 两小时试用路线

今天就想判断它值不值得，可以按五步走。第一，先在本机或 VPS 跑通 CLI。第二，配一个你网络和成本都能接受的模型。第三，选一个低风险但高重复的任务，比如资料整理、日报周报、测试结果巡检。第四，连续跑两到三次，观察它能不能把流程沉淀成 skill。第五，再决定是否接入 IM，或者把它放到长期在线的远程环境。

## Scene 07 - 结尾判断

我的判断是：Hermes Agent 值得先试，但不要立刻全量迁移。它真正抓住的是 agent 进入日常工作流后的关键问题：记忆、复用、异步执行和跨入口协作。先从低风险、高重复的工作流开始，等 skill 和记忆闭环跑通，再逐步扩大权限。这样它才可能从一个工具，变成你团队的自动化记忆层。
